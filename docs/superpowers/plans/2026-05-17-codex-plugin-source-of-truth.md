# Codex Plugin Source Of Truth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `codex/.codex/config.base.toml` preserve enabled Codex plugins so regenerated sessions do not require manual `codex-plugins-sync`.

**Architecture:** The checked-in base config should contain both marketplace declarations and plugin enablement entries. `codex-sync` will continue to regenerate the active `config.toml` from `config.base.toml`; plugin cache contents remain runtime state and stay ignored.

**Tech Stack:** TOML, shell verification, Git.

---

## File Structure

- Modify: `codex/.codex/config.base.toml`
  - Responsibility: durable Codex configuration source tracked in dotfiles.
- Generated by verification: `codex/.codex/config.toml`
  - Responsibility: active local Codex configuration regenerated by `codex-sync`.
  - Commit only if this repository already tracks the generated active config and the diff is limited to expected plugin enablement entries.

---

### Task 1: Promote Plugin Enablement To Base Config

**Files:**
- Modify: `codex/.codex/config.base.toml`
- Verify: `codex/.codex/config.toml`

- [ ] **Step 1: Inspect current plugin entries**

Run:

```bash
rg -n '^\[marketplaces\.|^\[plugins\.' codex/.codex/config.base.toml codex/.codex/config.toml
```

Expected: `config.toml` contains these plugin sections and `config.base.toml` does not:

```toml
[plugins."superpowers@superpowers-marketplace"]
enabled = true

[plugins."compound-engineering@compound-engineering-plugin"]
enabled = true
```

- [ ] **Step 2: Add plugin sections to the base config**

Edit `codex/.codex/config.base.toml` so the marketplace and plugin area reads:

```toml
[marketplaces.superpowers-marketplace]
# Load the Superpowers plugin marketplace from its Git repository.
source = "https://github.com/obra/superpowers-marketplace.git"
# Tell Codex to treat the marketplace source as a Git checkout.
kind = "git"

[marketplaces.compound-engineering-plugin]
# Load the Compound Engineering plugin marketplace from its Git repository.
source = "https://github.com/EveryInc/compound-engineering-plugin.git"
# Tell Codex to treat the marketplace source as a Git checkout.
kind = "git"

[plugins."superpowers@superpowers-marketplace"]
enabled = true

[plugins."compound-engineering@compound-engineering-plugin"]
enabled = true
```

- [ ] **Step 3: Regenerate active config**

Run:

```bash
codex-sync
```

Expected: command exits with status `0`.

- [ ] **Step 4: Verify regenerated config keeps plugin enablement**

Run:

```bash
rg -n '^\[plugins\."(superpowers@superpowers-marketplace|compound-engineering@compound-engineering-plugin)"\]|^enabled = true' codex/.codex/config.base.toml codex/.codex/config.toml
```

Expected: both files contain both plugin sections with `enabled = true`.

- [ ] **Step 5: Verify no plugin cache files are tracked**

Run:

```bash
git status --short -- codex/.codex/config.base.toml codex/.codex/config.toml codex/.codex/plugins
```

Expected: only `codex/.codex/config.base.toml` and, if regenerated content changed, `codex/.codex/config.toml` appear. No files under `codex/.codex/plugins` appear.

- [ ] **Step 6: Review the diff**

Run:

```bash
git diff -- codex/.codex/config.base.toml codex/.codex/config.toml
```

Expected: the diff only adds the two plugin enablement sections to the base config and preserves them in the generated active config if needed.

- [ ] **Step 7: Commit the implementation**

Run:

```bash
git add codex/.codex/config.base.toml codex/.codex/config.toml
git commit -m "fix(codex): persist enabled plugins in base config"
```

Expected: commit succeeds. If `codex/.codex/config.toml` has no diff after regeneration, `git add` leaves it unchanged and the commit includes only `codex/.codex/config.base.toml`.
