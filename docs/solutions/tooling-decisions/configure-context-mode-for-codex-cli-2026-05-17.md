# Configure Codex CLI hooks through config.base.toml

## Context

Codex CLI hook configuration in this dotfiles repo has three layers:

- Durable source config: `codex/.codex/config.base.toml`
- Generated runtime config: `codex/.codex/config.toml`
- Live Codex home wiring under `~/.codex`

`config.toml` is ignored and regenerated by `bin/.local/bin/codex-sync`, so
durable Codex settings belong in `config.base.toml`. That rule now includes
Codex hook registration.

The original context-mode setup followed the upstream `hooks.json` shape. That
created split-brain hook state: the atomic commit guard lived in TOML, while
context-mode hooks lived in `codex/.codex/hooks.json` and `~/.codex/hooks.json`.
The split became visible when Codex reported:

```text
PreToolUse hook (failed) error: hook exited with code 127
```

Investigation found two concrete failures:

- `hooks.json` invoked `context-mode hook codex pretooluse` as a bare command.
  It worked in an interactive shell, but Codex hooks run with a restricted
  non-interactive `PATH`; `context-mode` was installed through mise and was not
  resolvable there.
- The existing atomic commit hook command referenced
  `bash "$HOME/.codex/hooks/atomic-commits.sh"`, but
  `~/.codex/hooks/atomic-commits.sh` was not wired. The tracked script lived at
  `codex/.codex/hooks/atomic-commits.sh`.

Session history search found no relevant prior sessions for this specific
Codex hook failure.

## Guidance

Use `codex/.codex/config.base.toml` as the single tracked source for Codex hook
registration. Do not keep a separate tracked `codex/.codex/hooks.json` for this
setup.

Keep the atomic commit hook in TOML:

```toml
[[hooks.PreToolUse]]
matcher = "^Bash$"

[[hooks.PreToolUse.hooks]]
type = "command"
command = 'bash "$HOME/.codex/hooks/atomic-commits.sh"'
timeout = 10
statusMessage = "Checking atomic commit workflow"
```

Register context-mode hooks in the same TOML file. The command must be valid in
Codex's restricted hook environment, so include the mise install path, mise
shims, and ordinary system locations explicitly:

```toml
[[hooks.PreToolUse]]
matcher = "local_shell|shell|shell_command|exec_command|Bash|Shell|apply_patch|Edit|Write|grep_files|ctx_execute|ctx_execute_file|ctx_batch_execute|ctx_fetch_and_index|ctx_search|ctx_index|mcp__"

[[hooks.PreToolUse.hooks]]
type = "command"
command = 'PATH="$HOME/.local/bin:$HOME/.local/share/mise/shims:$HOME/.local/share/mise/installs/node/24/bin:/usr/local/bin:/usr/bin:/bin" context-mode hook codex pretooluse'
timeout = 10
statusMessage = "Indexing tool call context"
```

Use the same `PATH=... context-mode hook codex <event>` shape for:

- `PostToolUse` -> `posttooluse`
- `SessionStart` -> `sessionstart`
- `PreCompact` -> `precompact`
- `UserPromptSubmit` -> `userpromptsubmit`
- `Stop` -> `stop`

Delete `codex/.codex/hooks.json` after moving the hook registrations into TOML.
The live `~/.codex/hooks.json` path should be absent after migration.

Let `bin/.local/bin/codex-sync` own config generation and live wiring:

```sh
bin/.local/bin/codex-sync
```

The script should:

- Infer the dotfiles checkout from the script path when `DOTFILES` is unset.
- Generate `codex/.codex/config.toml` from `config.base.toml`.
- Wire `~/.codex/config.toml` to the generated config.
- Wire `~/.codex/hooks/atomic-commits.sh` to the tracked atomic hook.
- Remove only a managed stale `~/.codex/hooks.json` symlink.
- Refuse to remove an unmanaged `~/.codex/hooks.json`.
- Validate stale `hooks.json` before changing live symlinks, so failed
  migrations do not leave partial live state.
- Avoid live rewiring from non-primary worktrees unless `CODEX_HOME` is set or
  `CODEX_SYNC_LIVE=1`.

When testing from a worktree, either use an isolated `CODEX_HOME` or expect
generate-only behavior:

```sh
DOTFILES=/home/benkim0414/workspace/dotfiles/.worktrees/codex-hooks-single-source \
  bin/.local/bin/codex-sync
```

That command should generate the worktree config but not rewire real
`~/.codex` unless explicitly requested.

## Why This Matters

Codex hooks do not run in the same shell environment as an interactive terminal.
Any hook command that relies on shell startup files, mise activation, or an
interactive `PATH` can pass manually and fail inside Codex with exit `127`.

Keeping hook registration in `config.base.toml` also removes config drift. With
both TOML hooks and `hooks.json`, a future agent has to inspect two durable
files plus live `~/.codex` state to know what Codex will run. That ambiguity
made it easy to preserve stale context-mode wiring while fixing the atomic
commit hook, or vice versa.

`codex-sync` is the right place for live wiring because it already owns the
generated config boundary. The script can enforce safety rules that ad hoc
manual symlinks cannot:

- Fail clearly when tracked source files are missing.
- Refuse to overwrite non-symlink live files.
- Avoid deleting user-managed `hooks.json`.
- Avoid pointing real `~/.codex` at a disposable worktree.

## When to Apply

Use this pattern when:

- A Codex hook works interactively but fails with exit `127`.
- Hook commands depend on tools installed by mise, asdf, nvm, or shell startup
  files.
- Codex config is generated from tracked dotfiles.
- Multiple hook systems are competing to register the same Codex events.
- A feature worktree might accidentally rewrite the user's active Codex config.
- A legacy `hooks.json` migration needs to avoid deleting user-managed files.

Use the same source-of-truth rule for future Codex config changes: edit
`codex/.codex/config.base.toml`, then run `bin/.local/bin/codex-sync`.

## Examples

The old split-source setup registered context-mode in JSON:

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "context-mode hook codex pretooluse"
          }
        ]
      }
    ]
  }
}
```

That command depends on whatever `PATH` Codex gives the hook process. The TOML
replacement makes the hook source and executable lookup explicit:

```toml
[[hooks.PreToolUse.hooks]]
type = "command"
command = 'PATH="$HOME/.local/bin:$HOME/.local/share/mise/shims:$HOME/.local/share/mise/installs/node/24/bin:/usr/local/bin:/usr/bin:/bin" context-mode hook codex pretooluse'
timeout = 10
```

After sync, live paths should look like:

```text
~/.codex/config.toml -> /home/benkim0414/workspace/dotfiles/codex/.codex/config.toml
~/.codex/hooks/atomic-commits.sh -> /home/benkim0414/workspace/dotfiles/codex/.codex/hooks/atomic-commits.sh
```

`~/.codex/hooks.json` should be absent.

Verify the migration with both config tests and restricted-PATH reproductions:

```sh
codex/.codex/tests/test-codex-sync-hooks.sh
codex/.codex/tests/test-atomic-commits-hook.sh

printf '{}' | env -i HOME="$HOME" PATH="/usr/bin:/bin" sh -lc \
  'PATH="$HOME/.local/bin:$HOME/.local/share/mise/shims:$HOME/.local/share/mise/installs/node/24/bin:/usr/local/bin:/usr/bin:/bin" context-mode hook codex pretooluse'

printf '{"tool_input":{"command":"echo ok"}}' |
  env -i HOME="$HOME" PATH="/usr/bin:/bin" \
  bash "$HOME/.codex/hooks/atomic-commits.sh"
```
