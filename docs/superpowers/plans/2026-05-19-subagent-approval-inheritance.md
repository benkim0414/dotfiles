# Subagent Approval Inheritance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reduce repeated subagent approval prompts by inheriting durable Codex approval behavior while preserving prompts for sensitive operations.

**Architecture:** Keep `approval_policy = "on-request"` as the global safety boundary and add Codex's supported `approvals_reviewer = "auto_review"` setting with a narrow local policy. `codex/.codex/config.base.toml` remains the source of truth, `codex-sync` copies it to `codex/.codex/config.toml`, and the sync test verifies approval settings survive generation.

**Tech Stack:** Bash, TOML, Codex CLI config, `codex-sync`, shell-based regression tests.

## Task 2: Configure Scoped Auto Review For Approval Requests

**Files:**
- Modify: `codex/.codex/config.base.toml:13-21`
- Modify: `codex/.codex/config.toml` through `bin/.local/bin/codex-sync`

- [ ] **Step 1: Add the durable approval reviewer setting**

Edit the top approval block in `codex/.codex/config.base.toml` so lines 13-21 become:

```toml
# Allow reads plus writes inside the active workspace by default.
sandbox_mode = "workspace-write"
# Ask before leaving the sandbox or doing approval-sensitive work.
approval_policy = "on-request"
# Route approval requests through Codex's risk-based reviewer so subagents do
# not interrupt the user for routine low-risk operations.
approvals_reviewer = "auto_review"

# Avoid launching a GUI editor from Codex.
file_opener = "none"
# Enable live web search for current documentation and time-sensitive facts.
web_search = "live"
```

- [ ] **Step 2: Add the auto-review policy section**

Insert this section after `web_search = "live"` and before `[features]`:

```toml
[auto_review]
# Keep the reviewer narrow: routine repo work can proceed, boundary crossings
# still require explicit user intent or a durable scoped rule.
policy = """
Approve routine sandbox-compatible repository work: read-only commands, status
inspection, diffs, formatting checks, and test commands that write only inside
the active workspace or temporary directories.

Deny destructive commands, broad arbitrary shell approvals, network access,
credential access, writes outside configured workspace roots, and history
rewrites unless the user explicitly requested that exact operation in the
current conversation.

When an approval request includes a persistent prefix rule, approve only narrow
command-specific prefixes. Do not approve broad runtime prefixes such as bash,
python, node, ruby, perl, or sh.
"""
```
