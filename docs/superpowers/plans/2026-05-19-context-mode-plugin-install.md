# context-mode Plugin Install + remember Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land three atomic commits to `claude/.claude/settings.base.json` that remove the disabled `remember` plugin entry, alphabetize the remaining `enabledPlugins` keys, and enable the `context-mode` plugin; then verify via `claude-sync` and `/context-mode:ctx-doctor`.

**Architecture:** Pure JSON config edit. The `context-mode` plugin (`mksglu/context-mode`) auto-registers its MCP server and 5 hooks (PreToolUse, PostToolUse, UserPromptSubmit, PreCompact, SessionStart) via its own plugin manifest -- no manual hook registration in `settings.base.json` is required. Stowed config only toggles the `enabledPlugins` block. Marketplace registration and plugin install happen at runtime via slash commands the user executes in the REPL (Stow cannot manage runtime plugin state).

**Tech Stack:** GNU Stow, `jq`, `claude-sync` (deep-merge `settings.base.json` + `~/workspace/claude-skills/settings.overlay.json` -> `~/.claude/settings.json`), Claude Code plugin system.

**Worktree:** `worktree-context-mode-install` at `/Users/ben/workspace/dotfiles/.claude/worktrees/context-mode-install`. Already created. Spec landed in commits `405ebbd` + `1ac7ec4`. All edits below run in this worktree.
